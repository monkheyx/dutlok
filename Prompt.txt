I want you to help me design and build a private web application inspired by murlok.io, but only for my World of Warcraft guild.

This is NOT intended to be a public large-scale commercial site. It is for a single guild/community, so the system should be designed to be maintainable, reasonably fast, and practical rather than massively over-engineered. However, I still want it built in a professional way with clean architecture, strong organization, and room for future expansion.

Your job is to act as a senior full-stack architect and engineer. You are allowed to choose the codebase, framework, database, project structure, libraries, and implementation approach that best fits the requirements. Do not lock yourself into a stack too early if a better option emerges after analysis.

You should first analyze the project goals and then propose the best architecture, data model, and implementation plan before building. After the plan is approved internally by your own reasoning, begin implementation in a structured way.

The finished project should resemble the general usefulness of murlok.io, but tailored specifically to my guild, our characters, our activities, and our needs.

==================================================
CORE PROJECT GOAL
==================================================

Build a guild-focused World of Warcraft roster and build-analysis platform that:

1. Tracks guild members and their characters.
2. Pulls relevant character/build/profile data from Blizzard APIs or other appropriate official/allowed data sources.
3. Displays useful information about guild members in a clear dashboard.
4. Lets us analyze specs, builds, gear, performance-related choices, and guild composition.
5. Supports guild-specific views instead of public ladder-wide aggregation.
6. Is easy for our guild leadership/admins to manage.
7. Can grow over time if we later want more features.

This should feel like a polished internal guild tool, not just a rough script.

==================================================
HIGH-LEVEL PRODUCT VISION
==================================================

I want a site/app where guild members or guild leadership can:

- View the guild roster.
- See each character’s class, spec, role, faction, realm, item level, professions if available, PvE/PvP focus, and recent updates.
- View details for a specific character:
  - talents/builds
  - gear summary
  - embellishments / enchants / gems if available
  - specialization
  - stats/secondary distribution if available
  - raid / mythic+ / pvp relevance depending on what data is accessible
- Filter the roster by:
  - class
  - spec
  - role
  - active/inactive
  - main vs alt
  - raid team/group
- Mark or infer “main” characters vs alts.
- See guild composition summaries:
  - tanks / healers / dps
  - melee vs ranged
  - class/spec breakdown
  - armor type distribution
  - profession coverage
- Potentially identify weak coverage or missing utility in guild composition.
- Show recent changes or refreshes for characters.
- Optionally provide comparisons between guild members using the same spec.
- Optionally surface “recommended common patterns” from within our own guild data, or optionally from a broader reference source later if we choose.

Important:
This is for my guild, not the entire WoW player base, so the system should prioritize guild usefulness and data accuracy over giant-scale scraping.

==================================================
PRIMARY REQUIREMENTS
==================================================

The application should support these core areas:

1. Guild Roster
- Store guild members and associated characters.
- Support one member having multiple characters.
- Allow designation of main/alt.
- Show character status and summary data.
- Allow manual corrections if Blizzard data is incomplete or if guild-specific labels are needed.

2. Character Profiles
For each character, try to collect and display as much of the following as is reasonably available:
- character name
- realm
- region
- faction
- race
- class
- active spec
- role
- level
- item level
- talents/build import or talent summary
- gear summary
- stats / secondary stats if available
- profession data if available
- guild rank if available
- recent update timestamp
- avatar/render if available

3. Data Sync
- Pull data from Blizzard APIs or other official/permitted sources where appropriate.
- Build a sync process for:
  - initial guild import
  - refreshing all guild members
  - refreshing a single character
  - periodic scheduled updates
- Respect rate limits and use caching.
- Log sync results and failures.
- Build retry logic for transient failures.

4. Admin Controls
- Admins should be able to:
  - add/remove characters
  - map characters to guild members
  - mark main vs alt
  - flag active vs inactive
  - trigger refresh/sync
  - edit guild-specific metadata
- Admin should be simple and reliable.
- It does not need to be over-designed, but it should be functional and secure.

5. Guild Analytics
Provide dashboard-level views such as:
- total members
- active characters
- class/spec distribution
- role distribution
- mains vs alts
- profession coverage
- raid composition summaries
- optional comparison pages for same-spec characters
- optional “who can fill what role” views

6. Search / Filtering / Sorting
- Search by character name
- Filter by class/spec/role/main-alt/activity/team
- Sort by name, ilvl, recently updated, class, role, etc.
